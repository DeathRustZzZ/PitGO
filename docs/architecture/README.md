# Архитектурные диаграммы PitGO

## Назначение

Набор Mermaid-диаграмм, полученных обратной разработкой (reverse engineering)
текущего состояния кодовой базы. Диаграммы описывают **только то, что уже
реализовано в коде**, и служат картой для новых разработчиков, ревьюеров и
будущих сопровождающих.

## Что здесь есть и чего нет

Это не проектная документация и не целевая архитектура. Если компонент
описан в `docs/domain/`, но отсутствует в коде, его **нет** на диаграммах.
Единственный источник истины — исходный код.

Расхождения между задуманной моделью и реализацией вынесены в
[13_gaps.md](13_gaps.md) — читать его стоит вместе с остальными файлами,
иначе диаграммы могут создать впечатление большей завершённости, чем есть.

## Состояние на момент составления

- База: рабочее дерево после задач 001–005 Фазы 1
- Тестов: **100** — domain 50, application 20, infrastructure 13, backend 17
- Все репозитории — in-memory; PostgreSQL не реализован
- Фронтенд работает на моках и **не обращается** к Rust-бэкенду

Диаграммы устаревают быстрее кода. Если что-то расходится с исходниками,
источник истины — исходники; расхождение стоит завести как правку сюда.

## Условные обозначения

Единая легенда для всех файлов:

| Начертание | Значение |
|---|---|
| Сплошная линия `-->` | Вызов или зависимость, присутствующие в коде |
| Пунктир `-.->` | Связь заложена, но в коде не задействована |
| Сплошная рамка | Реализованный компонент |
| Пунктирная рамка | Компонент отсутствует в коде |

Любой пунктир на диаграмме — это то, чего **пока нет**. Если элемент
изображён сплошным, под ним есть конкретный файл и строка.
