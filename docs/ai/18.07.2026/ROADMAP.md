# ROADMAP — PitGO

Принципы: маленькие последовательные задачи; сначала стабилизация backend/domain/application; тесты — рано; инкрементальные улучшения вместо большого редизайна. Каждая задача имеет проверяемый результат. Порядок внутри фазы — по зависимостям и учебной ценности.

Файлы задач Фазы 1 лежат в [TASKS/](TASKS/). Задачи следующих фаз детализируются после завершения текущей фазы (пять файлов за раз — чтобы roadmap не устаревал).

---

## Фаза 1 — Стабилизация текущего среза backend

Цель фазы: честные ошибки, закрытый инвариант, доведённый Ownership-срез и закреплённый интеграционными тестами HTTP-контракт.

- [x] 001 — Закрыть дыру инварианта «одно активное владение»
- [x] 002 — Явная ошибка AlreadyExists в репозиториях
- [x] 003 — Семантические коды ошибок в API
- [x] 004 — GET-эндпоинт для VehicleOwnership
- [x] 005 — Интеграционные HTTP-тесты полного флоу

Зависимости: 003 опирается на 002 (маппинг новой ошибки); 005 выполняется последним (закрепляет контракт, стабилизированный в 001–004). 001 и 004 независимы от остальных.
